Sentriq API and Developer Guide

The Sentriq REST API lets you read your compliance data — controls, findings, evidence, and frameworks — and pull it into your own dashboards, data warehouse, or automation. API access is available on Scale and Enterprise plans.

Base URL and versioning

https://api.sentriq.site/v1

The API is versioned in the path (/v1). Breaking changes ship under a new version; additive changes (new fields, new endpoints) can appear within a version, so write clients that ignore unknown fields.

Authentication

Authenticate with an API token sent as a bearer header:

Authorization: Bearer <your_api_token>

Create and revoke tokens under Settings → API Tokens (admin only). Tokens are scoped read-only for compliance data. Treat tokens like passwords: store them in a secret manager, never in source control, and rotate them periodically.

Rate limits

Default limit is 120 requests per minute per token. Responses include X-RateLimit-Remaining and X-RateLimit-Reset headers. On 429 Too Many Requests, back off and retry after the reset time.

Core resources

Resource Endpoint Description
Frameworks GET /frameworks Frameworks enabled on your workspace and their status
Controls GET /controls Controls, owners, and pass/fail state
Findings GET /findings Open and resolved findings with severity
Evidence GET /evidence Evidence artifacts and the controls they support
Integrations GET /integrations Connected systems and last sync time

Examples

List open Critical and High findings:

GET /v1/findings?status=open&severity=critical,high
Authorization: Bearer <your_api_token>

Fetch a single control with its current evidence:

GET /v1/controls/CC6.1?include=evidence
Authorization: Bearer <your_api_token>

Pagination

List endpoints return up to 100 items per page. Use the cursor query parameter with the next_cursor value from the previous response to page forward. When next_cursor is null, you have reached the end.

Webhooks

Subscribe to events (for example, finding.opened, finding.resolved, control.failed) under Settings → Webhooks. Sentriq POSTs a signed JSON payload to your endpoint; verify the X-Sentriq-Signature header against your signing secret before trusting the body.

Errors

The API uses standard HTTP status codes. Error bodies are JSON with a stable code and a human-readable message:

{ "error": { "code": "rate_limited", "message": "Too many requests" } }

Support

API questions go to support@sentriq.site or in-app chat. Enterprise customers can request higher rate limits and a dedicated webhook retry queue through their CSM.
